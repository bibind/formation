# Planning demi-journee – API Platform Avance – Projet Marketplace

## Objectifs
- Detailer les sessions par demi-journee.
- Assurer l'alternance theorie/atelier.

## Contexte
5 jours = 10 demi-journees de 3h30 chacune.

## Planning
### Jour 1 matin
- Cadrage, architecture API Platform, DTO comme ressources.
- Atelier 1 : DTO Product/Category + mapping.

### Jour 1 apres-midi
- State Provider collection.
- Atelier 2 : Provider ProductCollection + pagination.

### Jour 2 matin
- State Processor creation Product.
- Atelier 3 : Processor ProductCreate + validation.

### Jour 2 apres-midi
- Commandes et regles metier stock.
- Atelier 4 : Processor OrderCreate + decrement stock.

### Jour 3 matin
- JWT, roles, security, securityPostDenormalize.
- Atelier 5 : Voter ProductVoter + regles d'acces.

### Jour 3 apres-midi
- Filtres standards + filtre custom.
- Atelier 6 : Filtre owner + Search/Order.

### Jour 4 matin
- Optimisation N+1, joins.
- Atelier 7 : Provider optimises.

### Jour 4 apres-midi
- Upload VichUploader.
- Atelier 8 : Endpoint upload image produit.

### Jour 5 matin
- Tests ApiTestCase et strategie.
- Atelier 9 : Tests collection, create, upload.

### Jour 5 apres-midi
- Etude de cas finale et evaluation.
- Atelier 10 : Mise en oeuvre complete + QCM sortie.

## Contraintes
- Respect des regles metier et contraintes techniques.

## Criteres de reussite
- Chaque demi-journee produit une progression mesurable.

## Pieges frequents
- Sous-estimer le temps des tests.

## Preuves audit possibles
- Planning signe avec horaires.
