# Objectifs pedagogiques Bloom – API Platform Avance – Projet Marketplace

## Objectifs
- Structurer les objectifs selon la taxonomie de Bloom.
- Garantir l'alignement avec ateliers et evaluation.

## Contexte
Les objectifs couvrent la progression de comprehension a creation.

## Objectifs Bloom
### Se souvenir
- Lister les composants d'API Platform (DTO, Provider, Processor).
- Nommer les roles JWT et leurs usages.

### Comprendre
- Expliquer la separation entre entites et ressources exposees.
- Decrire le cycle de resolution Provider/Processor.

### Appliquer
- Configurer des operations API sur un DTO.
- Implementer un Provider de collection avec filtres.

### Analyser
- Diagnostiquer un probleme N+1 sur une collection.
- Identifier les failles de securite dans un endpoint.

### Evaluer
- Comparer deux strategies de validation metier.
- Justifier l'usage d'un Voter vs securityPostDenormalize.

### Creer
- Construire une API Marketplace complete conforme aux contraintes.
- Produire une suite de tests ApiTestCase pour les scenarii principaux.

## Contraintes
- Chaque objectif est relie a un atelier et a un item d'evaluation.

## Criteres de reussite
- Les objectifs couvrent toutes les contraintes techniques et metier.

## Pieges frequents
- Rester au niveau "comprendre" sans actions verifiables.

## Preuves audit possibles
- Table d'alignement objectifs ↔ ateliers ↔ evaluation.
